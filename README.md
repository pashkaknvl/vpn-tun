# VPN-TUN

Учебный проект VPN-сервиса на Python с двумя независимыми частями:

- веб-интерфейсом для работы с пользователями;
- консольным VPN-клиентом и VPN-сервером с авторизацией через SQLite.

Это простой прототип, в котором показаны базовые идеи:

- хранение пользователей в базе данных;
- регистрацию и логин;
- поднятие `TUN`-интерфейса;
- передачу пакетов через `UDP`;
- простое шифрование пакетов через `XOR`.

## Стек

- `Python 3`
- `Flask`
- `SQLite`
- `PyYAML`
- `socket`
- `threading`
- `fcntl`, `os`, `struct` для работы с `TUN`
- `Docker / Dev Container` для удобного запуска Linux-окружения

## Идея проекта

Проект разделён на две логические части.

### 1. Веб-часть

Находится в каталоге `web/`.

Сейчас это отдельный интерфейс с регистрацией и логином через браузер. Он использует ту же базу `db.sqlite`, что и консольная часть, но напрямую не участвует в запуске VPN-туннеля.

### 2. VPN-часть

Находится в каталоге `vpn/`.

Это основной консольный сценарий для обычного пользователя:

- клиент запускается из терминала;
- пользователь выбирает `register` или `login`;
- данные отправляются на сервер по `UDP`;
- сервер проверяет пользователя в базе;
- после успешной авторизации обе стороны создают `TUN` и запускают туннель.

## Структура проекта

### Основные файлы

- `main.py`

- `db.sqlite`

### База данных

- `database/db.py`  
  Низкоуровневая работа с SQLite: `execute`, `fetchone`, `fetchall`, создание БД.

- `models/dbo.py`  
  Базовый класс для моделей с методами загрузки, вставки, обновления и удаления.

- `models/users.py`  
  Модель пользователей и поиск по имени.

- `models/roles.py`  
  Модель ролей.

- `models/permissions.py`  
  Модель прав.

### Генерация моделей

- `generators/codegen.py`  
  Генератор Python-моделей из схемы `config.yaml`.

### Веб-интерфейс

- `web/app.py`  
  Flask-приложение с веб-регистрацией и логином.

- `web/templates/`  
  HTML-шаблоны страниц.

- `web/static/login.css`  
  Стили интерфейса.

### VPN-часть

- `vpn/cli_auth.py`  
  Общий модуль консольной авторизации. В нём находятся хэширование пароля, ввод логина и пароля, сборка и разбор UDP-сообщений, проверка и регистрация пользователя в БД.

- `vpn/cli_vpn_client.py`  
  Консольный клиент. Запрашивает у пользователя действие `register/login`, отправляет данные на сервер и при успехе запускает туннель.

- `vpn/cli_vpn_server.py`  
  Консольный сервер. Слушает UDP-порт, принимает запрос на регистрацию или логин, проверяет базу и при успехе запускает туннель.

- `vpn/vpn_common.py`  
  Общие функции для VPN: создание `TUN`, `XOR`-шифрование, передача пакетов `TUN -> UDP` и `UDP -> TUN`.

## Как работает авторизация

Схема работы консольной части:

1. Пользователь запускает клиент.
2. Клиент спрашивает `register` или `login`.
3. Пользователь вводит имя и пароль.
4. Пароль хэшируется через `sha256`.
5. Клиент отправляет строку вида:
   `REGISTER|username|hash`
   или
   `LOGIN|username|hash`
6. Сервер принимает пакет и разбирает его.
7. При `register` сервер создаёт нового пользователя в таблице `Users`.
8. При `login` сервер сравнивает хэш с данными в базе.
9. Если всё успешно, сервер отвечает `OK|...`.
10. После этого обе стороны запускают туннель.

## Как работает туннель

После успешной авторизации:

- клиент создаёт `tun_client`;
- сервер создаёт `tun_server`;
- запускаются два потока;
- один поток читает пакеты из `TUN`, шифрует их через `XOR` и отправляет в `UDP`;
- второй поток принимает пакеты из `UDP`, дешифрует их и пишет обратно в `TUN`.

Важно: в проекте используется очень простое `XOR`-шифрование. Это учебная демонстрация, а не криптографически безопасный способ защиты трафика.

## Подготовка окружения

Для работы с `TUN` нужен Linux или Linux-контейнер.

Самый удобный вариант для этого проекта: запуск внутри `Dev Container`, потому что в `.devcontainer/devcontainer.json` уже добавлены:

- `NET_ADMIN`
- `/dev/net/tun`

Контейнер собирается на `Fedora 40`, а `Dockerfile` устанавливает:

- `python3`
- `pip`
- `sqlite`
- `flask`

## Запуск проекта

### 1. Инициализация базы данных

Если нужно создать таблицы по схеме:

```bash
python3 main.py
```

### 2. Запуск веб-части

```bash
python3 web/app.py
```

После этого веб-интерфейс будет доступен локально через Flask.

### 3. Запуск VPN-сервера

Из корня проекта:

```bash
sudo python3 -m vpn.cli_vpn_server 1194
```

### 4. Запуск VPN-клиента

Во втором терминале:

```bash
sudo python3 -m vpn.cli_vpn_client 127.0.0.1 1194
```

Дальше клиент предложит выбрать:

- `register`
- `login`

и попросит ввести `username` и `password`.

Если авторизация проходит успешно, клиент и сервер создают `TUN`-интерфейсы и запускают туннель.
