# VPN-TUN

Учебный VPN-сервис на языке Python. Проект демонстрирует полную цепочку работы сервиса: хранение учётных записей в базе данных, авторизацию пользователей и установление сетевого туннеля средствами ядра Linux.

Сервис включает два независимых компонента, использующих общую базу данных:

- `webapp` — веб-интерфейс на Flask, обеспечивающий регистрацию и вход через браузер.
- `vpnservice` — консольные клиент и сервер, выполняющие авторизацию по протоколу UDP и передачу данных через интерфейс `TUN`.

Проект является демонстрационным прототипом. Шифрование трафика реализовано операцией `XOR` и предназначено исключительно для иллюстрации принципа работы, а не для практического применения.

## Условия запуска компонентов

| Компонент | Поддерживаемые ОС | Требуемые права |
|-----------|-------------------|-----------------|
| Веб-интерфейс (`webapp`) | macOS, Linux, Windows | стандартные |
| Туннель (`vpnservice`) | Linux | `root` (доступ к `/dev/net/tun`, привилегия `NET_ADMIN`) |

Компонент туннеля использует устройство Linux `/dev/net/tun`, отсутствующее в macOS и Windows. Для его запуска в репозитории предусмотрен Dev Container на основе Fedora 40 (каталог `.devcontainer`).

## Запуск веб-интерфейса

```bash
python3 -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install flask pyyaml

python init_db.py
python -m webapp.server
```

После запуска интерфейс доступен по адресу `http://127.0.0.1:5000/`. Маршрут `/register` предоставляет форму регистрации, маршрут `/login` — форму входа. Данные учётных записей сохраняются в файл `db.sqlite`.

## Запуск туннеля

Запуск выполняется в среде Linux с правами `root` (в том числе в Dev Container). Сервер и клиент запускаются в отдельных терминалах.

Запуск сервера на UDP-порту:

```bash
sudo python3 -m vpnservice.server 1194
```

Запуск клиента с указанием адреса и порта сервера:

```bash
sudo python3 -m vpnservice.client 127.0.0.1 1194
```

Клиент запрашивает действие (`register` или `login`), а также имя пользователя и пароль. После успешной авторизации обе стороны создают интерфейсы `tun_client` и `tun_server` и устанавливают туннель.

## Архитектура

Приложение построено по слоевому принципу: каждый верхний уровень использует нижележащий и не зависит от деталей его реализации.

```
schema.yaml ──► init_db.py ──► core/connection.py ──► db.sqlite
                                      │
                            models/base.py
                                      │
              models/user.py · models/role.py · models/permission.py
                                      │
                  webapp/server.py        vpnservice/auth.py
```

Модуль `core/connection.py` отвечает за доступ к базе данных. Класс `Database` представляет собой потокобезопасную обёртку над `sqlite3`: доступ синхронизируется блокировкой `Lock`, строки возвращаются в формате `sqlite3.Row`. Класс создаёт таблицы на основании схемы и выполняет запросы методами `execute`, `fetchone` и `fetchall`.

Модуль `models/base.py` содержит базовый класс `DBO`, реализующий операции создания, чтения, изменения и удаления записей (`load`, `dbInsert`, `dbUpdate`, `dbDelete`) для произвольной таблицы. При изменении записи её предыдущая версия копируется в таблицу истории (`*_H`), при удалении — в таблицу удалённых записей (`*_D`); поля `Change_cnt`, `Last_change` и `Last_editor` обновляются автоматически.

Модули `models/user.py`, `models/role.py` и `models/permission.py` описывают конкретные сущности. Каждый из них задаёт перечень полей и имя таблицы, наследуя логику работы с базой от класса `DBO`. Класс `User` дополнительно реализует поиск по имени пользователя (`find_by_username`).

Модуль `tools/model_generator.py` формирует файлы моделей на основании описания таблиц в `schema.yaml`.

Каталоги `webapp` и `vpnservice` представляют два интерфейса к общей базе данных: веб-интерфейс и консольный сетевой интерфейс соответственно.

## Протокол авторизации

Клиент и сервер обмениваются текстовыми сообщениями по протоколу UDP. Пароль передаётся в виде хеша `sha256` и не пересылается в открытом виде.

Формат запроса клиента:

```
REGISTER|<username>|<sha256(password)>
LOGIN|<username>|<sha256(password)>
```

Формат ответа сервера:

```
OK|<сообщение>
ERR|<причина>
```

Сервер разбирает полученный пакет, проверяет данные в таблице `Users` и формирует ответ. Установление туннеля выполняется только после получения ответа `OK`.

## Передача данных через туннель

После авторизации каждая сторона создаёт интерфейс `TUN` и запускает два потока, определённых в `vpnservice/tunnel.py`:

- поток `TUN → UDP` считывает IP-пакеты из интерфейса, применяет операцию `XOR` и отправляет их в сокет;
- поток `UDP → TUN` принимает пакеты из сети, снимает `XOR` и записывает их в интерфейс.

Таким образом трафик одной стороны инкапсулируется в UDP, передаётся по сети и восстанавливается на стороне приёма.

## Используемые технологии

Python 3, Flask, SQLite, PyYAML, стандартные модули `socket`, `threading`, `fcntl`, `struct`, `os` для работы с интерфейсом `TUN`, а также Docker и Dev Container для подготовки окружения.

```

│
├── core/
│   └── connection.py       класс Database, доступ к SQLite

├── models/

│   ├── role.py             модель ролей
│   └── permission.py       модель прав
│
├── tools/

│
├── webapp/
│   ├── server.py           маршруты, сессии, регистрация и вход
│   ├── templates/          home.html, login.html, register.html
│   └── static/style.css    оформление интерфейса
│
├── vpnservice/
│   ├── auth.py             протокол авторизации и работа с базой

│

```
